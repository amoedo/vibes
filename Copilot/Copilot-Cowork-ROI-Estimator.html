How this calculator works

This tool keeps Microsoft's original Copilot Cowork credit-consumption model and extends it into a directional ROI comparison against the cost of doing the same work with human labour. Work through the steps top to bottom; every figure is editable and the page recalculates instantly.

The calculation, step by step

  • Steps 1–2: for each archetype you set the number of users and how many Light / Medium / Heavy prompts each runs per month.
  • Step 3: each prompt type consumes a fixed number of Copilot Credits (125 / 500 / 1,200 by default). Credits convert to money at the list price — 100 credits = 1 currency unit, i.e. $0.01 per credit, matching Microsoft's pay-as-you-go rate.
  • Step 4: monthly credits = Σ (users × credits per user); budget = credits ÷ 100. This is the original Microsoft output, plus an annualized line.
  • Step 5: a fully-loaded hourly cost per archetype = salary × (1 + employer burden) ÷ productive hours per year.
  • Step 6: you estimate how long a human would take to do the equivalent of each task type, which yields the human-equivalent hours and a per-task break-even.
  • Step 7: human labour value = human hours × hourly cost × value realization %. ROI-ish = (labour value − Cowork cost) ÷ Cowork cost.

Why the Value Realization % matters most

This single assumption is the biggest driver of the result, and the easiest to overstate. It answers: of the human time a task would have taken, how much do you actually recover as real, redeployable value?

Rarely is it 100%. AI output usually needs review, correction, or context that the original worker still supplies; saved minutes are often scattered and not recaptured as productive work; and not every "task" the AI does would truly have been done by a person. A realization of 50% (the default here) says half the theoretical time saving converts into genuine value — a deliberately conservative, defensible starting point. Move it up only if you can evidence it. Use the sensitivity band in Step 7 (50 / 75 / 100%) when presenting, rather than betting on a single optimistic number.

Adoption factor

Scales how many licensed users are actually active. It moves cost and value together, so absolute totals reflect a realistic ramp while the ROI ratio is unchanged.

Benchmarks, methodology & references

Rule of thumb: a fully-loaded knowledge-worker cost is €40–€70/hour in Europe/Spain-style settings, roughly $60–$95/hour in the US, and $5–$20/hour for an internal offshore employee in India. The default presets below feed Step 5; all values remain editable.

Blended-rate shortcuts & sensitivity

Suggested blended rates for AI ROI: €50/h conservative European knowledge worker; €75/h if the population skews senior tech / consulting / legal / finance / management (Step 5 buttons). Because the result is so sensitive to the Value Realization %, present the Step 7 sensitivity band (50 / 75 / 100%) rather than a single point estimate. The default realization is set to a deliberately conservative 50%.
